# TSBCV
1、面向草原承包合同中的空白一致性校验任务，提出基于两阶段语义建模的空白一致性校验模块TSBCV，以解决模板填空式合同中相同语义空白项填写不一致、不同语义空白项错误填写相同内容等问题。该模块通过建模空白项与局部上下文之间的语义关系，以及不同空白字段之间的全局对应关系，实现对合同内部填写冲突和逻辑不一致问题的自动检测，从而提升合同文本结构一致性审查的准确性与效率。

2、从任务本身来看，空白一致性校验并不是简单的字符串匹配问题。很多情况下，相关字段分散在不同条款中，表述方式并不完全一致，有些字段还需要结合局部上下文才能判断其真实语义角色。也就是说，系统不仅要识别“填了什么”，还要判断“该位置本应填写什么”以及“它与文中其他位置是否属于同一语义槽位”。如果依赖人工逐项核对，审查效率较低，漏检和误判也难以避免；如果仅依赖关键词或规则匹配，又难以处理跨句、跨段乃至跨条款的语义对应关系。
如图上图所示，红色区域表示不同语义空白项填写了相同内容，蓝色区域表示语义相同的空白项填写了不同内容。此类问题未必直接表现为高风险条款，但会削弱合同文本的规范性和可执行性，并增加后续履约、审查和争议处理中的解释成本。

3、整体包含三个单元：空白语义表示单元、局部上下文感知单元以及全局关系更新与一致性判定单元。空白语义表示单元负责对合同文本中的空白字段及其上下文进行统一编码，生成初始字段表示；局部上下文感知单元围绕空白附近的关键词和约束描述提取局部证据，使模型更准确地识别该空白在当前位置承担的语义角色；全局关系更新与一致性判定单元则在合同全文范围内建模不同空白字段之间的对应关系与区别关系，得到更完整的字段表示，并在此基础上完成最终的一致性关系判定。通过三个单元的协同作用，TSBCV能够同时利用空白的局部语义线索与文档级结构信息，实现对合同内部填写冲突与逻辑不一致问题的自动识别。

4、该案例表明，TSBCV不仅能够识别空白字段在局部语境中的语义角色，还能够在合同全文范围内建立跨条款字段之间的对应关系，从而较准确地发现草原承包合同中的填写冲突和结构性不一致问题。
